\section{Introduction}
The Snake Game, themed like Minecraft, is an upgraded version of the infamous Snake Game played in most arcades and mobile phones. The website based game works in three layers. It uses HTML, CSS, and JavaScript for game display, control, and logic. The next layer is Python, using the flask external library. The final, backend layer is bash script for display of previous data on query.

\section{External Libraries}
\subsection{Flask}

Flask is a lightweight WSGI web application framework\cite{flaskwebsite}. Of the many uses flask has, we have made use of flask for HTML templating, routing and handling requests. In app.py, \textit{Falsk}, \textit{send\_from\_directory}, \textit{request} and \textit{jsonify} are being imported from flask. 

\subsubsection*{Flask}

Flask is the core application object. It creates a web application instance, assigned to the variable \textit{app}, that can handle, requests, routing and responses. So \textit{app} is the representation of the actual working website we created.

\subsubsection*{send\_from\_directory}

send\_from\_directory is a Flask function that directly serves the html file from static folder and sends it to the browser as a response. The Python program executes the run method of \textit{app,} which starts a server and waits for incoming requests. When the HTTP link is opened, the browser sends a GET / request. This triggers the function index, which calls send\_from\_directory. Flask then looks inside the static, locates index.html, and returns it to the browser without using any templating engine.

\subsubsection*{request}

request is a Flask object that contains data send by the client, this includes the JSON body. In this code we use the \textit{get\_json} method of the request object which returns a python dictionary from the incoming JSON data.

\subsubsection*{jsonify}

jsonify is a function in flask that converts the python dictionary into a JSON response and sends it to the client that sent the POST request.

\section{Features}
\subsection{Title Screen}
The title screen is the first screen visible when the brower is opened and navigated to \textit{http://127.0.0.1:5000}. This screen contains the game logo and two button; the instructions button and the login button. They open the instructions modal and username entry modal respectively.

\subsection{Instruction Modal}
This modal displays the rules for the Snake Game. It is made with a minecraft book background. There are two buttons, one that takes the user to the username entry modal and other that closes the modal.

\subsection{Username Entry Modal}
This displays a form that allows the user to enter a username and only then play the game. It also has button to close the modal and a button that directs it to the instruction modal.

\subsection{Game Controls}
The Snake Game involves controlling the snake to move towards the food. This controlling can be done through both the arrow keys and WASD keys on the keyboard.

\subsection{Food types}
There are multiple types of food in the game.
\subsubsection*{Carrot \raisebox{-0.1cm}{\includegraphics[width=0.6cm]{Carrot_JE3_BE2.png}}}
This is a simple food type that adds one point to the score and one unit length to the snake. It has a 60\% probability to appear.

\subsubsection*{Pumpkin Pie \raisebox{-0.1cm}{\includegraphics[width=0.6cm]{Pumpkin_Pie_JE2_BE2.png}}}
Pumpkin pie is a food type with extra health. It adds three points to the score and adds length three to the snake. It has a 25\% 

\subsubsection*{Sweet Berries \raisebox{-0.1cm}{\includegraphics[width=0.6cm]{Sweet_Berries_JE1_BE1.png}}}
Sweet Berries are booster foods. They increase the speed by a factor of 2 for 10 seconds. If a sweet berry is consumed while the speed is already doubled, it simply restarts the timer from 10. It does not add to the score or change length of body. It has a 10\% probability of appearing.

\subsubsection*{Golden Apple \raisebox{-0.1cm}{\includegraphics[width=0.6cm]{Golden_Apple_JE2_BE2.png}}}
Golden Apples are immunity foods that provide immortality for 10 seconds. The snake does not die when it collides with its own body when immune and passes through appearing on the opposite wall if it runs into them. If a golden apple is consumed while the snake is already immune, it simply restarts the timer from 10. It does not add to the score or change length of body. It has a 5\% probability of appearing.

\subsection{Timers}
There are two timers in the game: \textbf{Immunity Timer} and \textbf{Boost Timer}. Both of these appear when the respective food is consumed and counts down from 10 to 0 unless the food is eaten again within those 10 seconds. The immunity timer appears to the top right of the game canvas and the boost timer appears to the top left corner.

\subsection{Game Over Screen}
The game ends when the snake hits the wall or itself. Under such situations, the game over screen is displayed. It has a red overlay with display details about the game score, username, cause of death, duration game was played. It also has two buttons; \textbf{Respawn} and \textbf{Title Screen}; Respawn allows one to replay the game with the same username. To play with a different username, one can go back to the Title Screen.

\subsection{Bash Query}
 admin.sh performs various operations on the data flat history.txt file using various bash tools. \textit{admin.sh}, designed for backend data management. This tool allows us to interact directly with \textit{history.txt} through a menu-driven interface in the terminal without requiring the Flask server to be active. Key features include:
\begin{itemize}
    \item \textbf{User Filtering:} Allows us to select a specific username; once selected, all subsequent statistics and views are filtered to show only that user's history.
    \item \textbf{Analytics:} Calculates things like mean scores, average survival time, and the percentage of deaths caused specifically by wall or collision with body.
    \item \textbf{Deletion by filtering:} Provides functionality to delete specific entries username or entries that don't follow a particular format.
    \item \textbf{Log Rotation:} Implements a rotation system that archives and compresses (Gzip) the current history file with a date extension and leaves the log file with just 10 recent entries.
\end{itemize}

\section{Design and Architecture}
The project works in three layers. Frontend: \textbf{Javascript}, \textbf{Python: Flask} and backend: \textbf{Bash Script}.

HTML, CSS and JavaScript make up the frontend of the game, everything that is seen on the screen during the game is controlled and coded in web only. The game logic is in JS. 

Flask library in python is used to serve index.html from static and send it to the browser. When \textit{python3 app.py} is run, it creates a server and waits for requests. When the link is clicked opened, the browser sends a GET / request to flask and this calls the send\_from\_directory function. Flask also takes the json object sent by the \textit{saveScore} function in JS and
and writes it into \textit{history.txt}. It also returns a json object that shows the success and failure of the data being saved.

When data is saved into \textit{history.txt}, it is of the format 

\begin{center}
    [2025-06-12 14:35:02] Herobrine \textbar 42 \textbar WALL \textbar 85
\end{center}

This data can be accessed using the bash query feature. This layer interacts with history.txt only.

\section{Code Logic}
\subsection{Game Logic}
The game logic has a few key ideas behind it

\subsection{Query Logic}
    \subsubsection*{Choice-Specific Logic}

    \begin{description}
        \item[Choice 1 (User Selection):] The script reads user input into a global variable "local". If the input is blank, the variable is cleared, resetting the scope to the entire file for all other menu options.
        \item[Choice 2 (Paginated View):] The output of \textit{data()} is piped into \textit{less}. This allows us to scroll through large log files, search for strings, and navigate without cluttering the terminal .

        \item[Choice 3 (Analytical Processing):] The filtered data is passed to \textit{awk} with the pipe (\textbar) defined as the field separator (\verb|-F'|'|).
        \begin{itemize}
            \item \textbf{Accumulation:} As \textit{awk} iterates through lines, it increments a counter (\textit{n}) and adds values from field \$2 (Score) and \$4 (Time) to running totals.
            \item \textbf{Pattern Matching:} It checks field \$3 for the string "wall" using a regular expression match string. If found, a "wall death" or "body" counter is incremented.
            \item \textbf{Final Calculation:} In the \verb|END| block, the script performs floating-point division to produce means and percentages, formatted to one decimal place via \textit{printf}.
    \end{itemize}

    \item[Choice 4 (Deletion):] The script uses \textit{grep -v} (inverse match) to stream every line \textbf{except} the one matching the target into a temporary file. This handles both specific username/timestamp deletions and "invalid" line cleaning. The \verb|mv| command then replaces the database .

    \item[Choice 5 (Cyclic Rotation):] This follows a "Backup-then-Truncate" logic. First, the \textit{tar} utility creates a compressed file. The filename is generated dynamically using \verb|date +%Y%m%d| to prevent overwriting same-day backups. Then \textit{tail} command keeps only the 10 most recent entries.

    \item[Choice 6 (Columnar Sorting):] The logic utilizes the \textit{sort} utility with specific flags. For scores, it uses \verb|-k2,2 -nr|, which tells the system to look strictly at the second field, treat it as a number, and sort in reverse (highest first). For users, it uses \verb|-t']' -k2|, splitting the line at the closing bracket of the timestamp to isolate and sort by the username field.
\end{description}
